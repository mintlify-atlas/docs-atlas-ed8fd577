The `/translate` endpoint performs a concept lookup against the loaded mappings, returning the matched ICD-11 code for a given NAMASTE source code. This endpoint is rate limited to **30 requests per minute** per IP address.

## Endpoint

**Method:** `POST`  
**Path:** `/translate`  
**Auth:** None (demo mode)  
**Rate Limit:** 30 requests per minute

## Request Body

<ParamField body="sourceCode" type="string" required>
  The NAMASTE code to translate. Must be a string value (e.g. `"NAM-001"`).
</ParamField>

<ParamField body="sourceSystem" type="string" required>
  The coding system of the source code. Must be `"NAMASTE"`.
</ParamField>

## Response

<ResponseField name="sourceCode" type="string">
  The NAMASTE code that was submitted for translation.
</ResponseField>

<ResponseField name="targetCode" type="string">
  The matched ICD-11 code (e.g. `"TM2-001"` or `"BM-001"`).
</ResponseField>

<ResponseField name="targetSystem" type="string">
  The ICD-11 system the target code belongs to. Either `"ICD-11 TM2"` or `"ICD-11 Biomedicine"`.
</ResponseField>

<ResponseField name="equivalence" type="string">
  The mapping type. One of `equivalent`, `equal`, `wider`, or `subsumes`.
</ResponseField>

<ResponseField name="confidence" type="number">
  A float from `0.0` to `1.0` representing the accuracy of the mapping.
</ResponseField>

## Error Responses

| Status | Condition |
|---|---|
| `400 Bad Request` | `sourceCode` or `sourceSystem` is missing or empty ŌĆö returns `"Validation failed"` with a `details` array. |
| `400 Bad Request` | `sourceCode` or `sourceSystem` is present but not a string type ŌĆö returns `"Invalid input format"`. |
| `404 Not Found` | No mapping exists for the provided `sourceCode` / `sourceSystem` pair. |

A `400` from missing or empty fields includes a `details` array listing each validation failure:

```json 400 ŌĆö Validation Error
{
  "error": "Validation failed",
  "details": [
    "Missing required field: sourceCode"
  ]
}
```

A `400` from a non-string value returns a flat error object (no `details` array):

```json 400 ŌĆö Invalid Input Format
{
  "error": "Invalid input format"
}
```

```json 404 ŌĆö No Mapping Found
{
  "error": "No mapping found"
}
```

<CodeGroup>


```javascript JavaScript
  method: 'POST',

  },

    sourceSystem: 'NAMASTE'
  })

const result = await response.json();

```

</CodeGroup>

```json Response
  "targetCode": "TM2-001",
  "targetSystem": "ICD-11 TM2",
  "equivalence": "equivalent",
  "confidence": 0.9
}
```

## Validation

Both `sourceCode` and `sourceSystem` are validated in two sequential steps before the lookup is performed:

1. **Presence check** ŌĆö Both fields must exist in the request body and must not be empty. Failures here return `{"error": "Validation failed", "details": [...]}` with an array that names every missing field.
2. **Type check** ŌĆö Both values must be strings. Passing a number, `null`, or any non-string type triggers a separate `{"error": "Invalid input format"}` response with no `details` array.

The two-step design means a caller may receive either error shape depending on which check fails first.

<Note>
  Currently only the **NAMASTE ŌåÆ ICD-11** translation direction is supported. Reverse lookups (ICD-11 ŌåÆ NAMASTE) are not yet implemented and will return a `404` if attempted.
</Note>
